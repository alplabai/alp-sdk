# E1M-AEN401 M55/Zephyr USB Host Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the M55/Zephyr USB host stack for E1M-AEN401 (on-SoC DWC2): metadata, a DWC2-host `uhc` driver skeleton, the `alp_usb_host_*` backend wiring, and a mass-storage host example — all compiling no-board, with live bring-up bench-gated.

**Architecture:** Component 0 grounds E4's USB controller into `e4.json` (gate). Component 1 adds a DWC2-host `uhc` driver in the alp-sdk Zephyr module (`zephyr/drivers/usb/uhc/`) — the full `uhc_api` op table + Kconfig/CMake/DT-binding, with register-level bring-up grounded from the in-tree `udc/dwc2` sibling and marked `TODO(aen401-bench)`. Component 2 wires `z_host_*` to Zephyr's `usbh_*` stack. Component 3 generates the AEN401 M55 board + a USB-storage example. Everything is validated by compiling for `alp_e1m_aen401_m55_hp`.

**Tech Stack:** Zephyr 4.4 (`usbh`/`uhc`), Synopsys DWC2 USB IP, C (Zephyr driver style), `py -3.14` for metadata/board-gen, west, WSL.

## Global Constraints

- Never invent register offsets/addresses — transcribe DWC2 host registers from the in-tree `udc/dwc2` device driver (shared Synopsys core) / the Alif HWRM, else `TODO(aen401-bench)`.
- New C under `zephyr/**` follows Zephyr driver style (the clang-format-22 gate EXCLUDES `zephyr/**`); C under `src/**` obeys clang-format-22 (verify with WSL `~/.local/bin/clang-format` 22.1.5).
- "Alp Lab" spelling; no `Co-Authored-By: Claude` footer. `py -3.14` for Python. No binaries in git.
- Branch `feat/aen401-usb-host` (off `dev`) → PR to `dev`.
- Zephyr workspace for builds: `~/zephyrproject` (Zephyr 4.4.0, arm-zephyr-eabi SDK) in WSL; drive west via a `.sh` through `MSYS_NO_PATHCONV=1 wsl bash /mnt/c/.../script.sh`. The alp-sdk module is pulled in via `EXTRA_ZEPHYR_MODULES=<repo>`.
- The DWC2-host driver is a COMPILE-ONLY skeleton: every op whose behavior can't be verified no-board returns a defined status and carries a `TODO(aen401-bench)` marker. Do NOT claim runtime correctness; do not fabricate a register bring-up sequence as if validated.

```c
#if defined(CONFIG_USB_HOST_STACK)
#include <zephyr/usb/usbh.h>

/* One host controller, bound to the board's alif,dwc2-uhc node. */
USBH_CONTROLLER_DEFINE(alp_usbh, DEVICE_DT_GET(DT_CHOSEN(zephyr_uhc)));

static alp_status_t z_host_open(alp_usb_host_state_t *st, alp_capabilities_t *caps_out)
{
	caps_out->flags = 0u;
	if (usbh_init(&alp_usbh) != 0) {
		return ALP_ERR_IO;
	}
	st->be_data = &alp_usbh;
	return ALP_OK;
}
static alp_status_t z_host_enable(alp_usb_host_state_t *st)
{
	struct usbh_context *uhs = st->be_data;
	return (uhs && usbh_enable(uhs) == 0) ? ALP_OK : ALP_ERR_IO;
}
static alp_status_t z_host_disable(alp_usb_host_state_t *st)
{
	struct usbh_context *uhs = st->be_data;
	return (uhs && usbh_disable(uhs) == 0) ? ALP_OK : ALP_ERR_IO;
}
static void z_host_close(alp_usb_host_state_t *st)
{
	struct usbh_context *uhs = st->be_data;
	if (uhs) {
		usbh_shutdown(uhs);
		st->be_data = NULL;
	}
}
#else
/* (existing NOSUPPORT stubs unchanged) */
#endif
```

```c
/*
 * Copyright 2026 Alp Lab AB
 * SPDX-License-Identifier: Apache-2.0
 *
 * usb-host-storage -- open the USB host role via the portable
 * <alp/usb.h> surface and enumerate an attached USB mass-storage
 * device on the E1M-AEN401 M55.  Bring-up + live enumeration are
 * bench-gated (the DWC2-host driver is a skeleton); this builds the
 * host path end-to-end.
 */
#include <stdio.h>
#include <zephyr/kernel.h>
#include <alp/usb.h>

int main(void)
{
	printf("== usb-host-storage (E1M-AEN401 M55) ==\n");

	alp_usb_host_t *host = alp_usb_host_open();
	if (host == NULL) {
		printf("alp_usb_host_open failed (NOSUPPORT until the DWC2-host driver lands)\n");
		return 1;
	}
	if (alp_usb_host_enable(host) != ALP_OK) {
		printf("alp_usb_host_enable failed\n");
		alp_usb_host_close(host);
		return 1;
	}
	printf("USB host enabled -- attach a mass-storage device (enumeration is bench-gated)\n");
	/* TODO(aen401-bench): on real silicon, wait for enumeration + mount
	 * the MSC LUN via Zephyr's usb_host MSC class, then list the root. */
	k_sleep(K_SECONDS(2));
	alp_usb_host_disable(host);
	alp_usb_host_close(host);
	return 0;
}
```
